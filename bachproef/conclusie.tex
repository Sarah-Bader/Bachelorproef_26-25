In deze bachelorproef werd onderzocht in welke mate statische analyse en Mermaid-diagrammen kunnen helpen bij het leren van HLASM door beginnende ontwikkelaars. 

De centrale onderzoeksvraag luidde:

\begin{quote}
    \itshape
    Hoe effectief versnellen statische analyse en Mermaid-diagrammen het leerproces van HLASM bij beginners?
\end{quote}

Om deze vraag te beantwoorden werd eerst een literatuurstudie uitgevoerd naar mainframes, HLASM en de moeilijkheden die beginners ervaren bij assemblytalen. Daarna werd een analyse gemaakt van HLASM om te bepalen welke onderdelen belangrijk zijn voor de control-flow. Op basis hiervan werd een proof-of-concept ontwikkeld die HLASM-code omzet naar een diagram in Mermaid.js.

Uit de evaluatie blijkt dat de visualisatie vooral helpt om sneller een overzicht te krijgen van de structuur van een programma. De deelnemers konden makkelijker zien hoe verschillende delen van de code met elkaar verbonden zijn en waar vertakkingen plaatsvinden.

Tegelijk werd duidelijk dat deze visualisatie niet voldoende is om de code volledig te begrijpen. De deelnemers gaven aan dat het nog steeds nodig is om de code zelf te lezen en te analyseren. De tool versnelt dus niet het volledige leerproces, maar helpt vooral bij het begrijpen van de globale structuur.

De tool wordt daarom best gezien als een aanvullend hulpmiddel, en niet als een vervanging van manuele analyse. Vooral bij grotere programma’s biedt de visualisatie een duidelijke meerwaarde, omdat het moeilijker is om daar zelf snel een overzicht van te krijgen. Bij kleinere programma’s is deze meerwaarde beperkter.

Er moet rekening worden gehouden met de beperkingen van dit onderzoek. De proof-of-concept ondersteunt slechts een beperkt deel van HLASM en houdt geen rekening met complexere constructies zoals macro’s of indirecte sprongen. Daarnaast was de testgroep klein, waardoor de resultaten niet volledig veralgemeend kunnen worden.

Ondanks deze beperkingen toont dit onderzoek aan dat een eenvoudige visualisatietool nuttig kan zijn bij het analyseren van HLASM-code. Dit is relevant in de praktijk, waar het begrijpen van bestaande code vaak moeilijk is door een gebrek aan documentatie.

Voor toekomstig werk kan de tool verder uitgebreid worden met ondersteuning voor meer instructies en complexere situaties. Ook kan onderzocht worden hoe deze visualisatie geïntegreerd kan worden in bestaande ontwikkelomgevingen. Daarnaast kan een grotere evaluatie bijkomende inzichten geven.

Samengevat kan gesteld worden dat statische analyse in combinatie met een visuele voorstelling helpt om de structuur van HLASM-programma’s sneller te begrijpen. De tool ondersteunt het leerproces, maar vervangt het niet.
