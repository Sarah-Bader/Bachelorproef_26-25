\section{Architectuur van de applicatie}

De proof-of-concept werd geïmplementeerd als een webapplicatie die HLASM-broncode omzet naar een control-flowdiagram in Mermaid.js-syntax.
Omdat het een webapplicatie is, hoeft de gebruiker niets te installeren.  
De tool kan meteen gebruikt worden via de browser. Dit maakt het eenvoudig om de applicatie snel te testen en te gebruiken.

De werking van de applicatie bestaat uit drie stappen:

\begin{enumerate}
    \item \textbf{Parsing}: de ingevoerde HLASM-broncode wordt opgesplitst in individuele instructies. Per regel worden label, mnemonic en operanden geïdentificeerd op basis van de kolomgebaseerde structuur.
    
    \item \textbf{Control-flow analyse}: op basis van de gedetecteerde instructies wordt bepaald hoe de uitvoering van het programma kan verlopen.
    
    \item \textbf{Visualisatie}: de structuur wordt omgezet naar Mermaid.js-syntax, die vervolgens wordt omgezet naar een flowchart.
\end{enumerate}

\subsection{Stap 1: Parsing}

In deze eerste stap wordt de HLASM-code verwerkt tot een lijst van instructies.
Lege lijnen en commentaar (lijnen die starten met een *) worden genegeerd.

Voor elke lijn worden drie onderdelen bepaald:
\begin{itemize}
    \item het label (indien aanwezig),
    \item de mnemonic (de instructienaam),
    \item het eerste operand (argument).
\end{itemize}

De parser maakt gebruik van de vaste kolomstructuur door specifieke delen van de lijn uit te lezen op vaste posities.

\begin{lstlisting}[language=JavaScript, caption={De HLASM parser.}]
function parseInstructions(hlasmCode) {
    const instructions = [];
    
    hlasmCode.split('\n').forEach((line, index) => {
        if (!line || line.trim() === "" || line.trim().startsWith("*"))
        return;
        
        const padded   = line.padEnd(80, " ");
        const label    = padded.substring(0, 8).trim();
        const mnemonic = padded.substring(9, 14).trim().toUpperCase();
        
        if (!mnemonic)
        return;
        
        const operandField = padded.substring(15, 71).trim();
        
        const operandMatch = operandField.match(/^([^\s']+|'[^']*')+/);
        const fullOperand  = operandMatch ? operandMatch[0] : "";
        const firstOperand = fullOperand.split(',')[0].trim();
        
        instructions.push({
            lineNumber: index + 1,
            raw: `${label.padEnd(8)} ${mnemonic.padEnd(5)} ${fullOperand}`,
            label,
            mnemonic,
            operand: firstOperand
        });
    });
    
    return instructions;
}
\end{lstlisting}

\subsection{Stap 2: Control-flow analyse}

%simpele uitleg - opdeling in blokken

\begin{lstlisting}[language=JavaScript, caption={Het fragment van die de instructies opdeeld in blokken.}]
function buildBlocks(instructions) {
    let blockId = 0;
    const blocks = [];
    let currentBlock = createBlock(0);
    
    instructions.forEach((instr, i) => {
        if (instr.label && currentBlock.lines.length > 0) {
            blocks.push(currentBlock);
            currentBlock = createBlock(++blockId);
        }
        
        currentBlock.lines.push(instr);
        
        if (instr.label) {
            currentBlock.labels.push(instr.label);
        }
        
        const isBranch = isBranchInstruction(instr.mnemonic);
        const isEnd = instr.mnemonic === "END";
        
        if (isBranch || isEnd) {
            currentBlock.exit = instr;
            blocks.push(currentBlock);
            currentBlock = createBlock(++blockId);
        } else if (instructions[i + 1] && instructions[i + 1].label) {
            currentBlock.exit = { mnemonic: "SEQUENTIAL", operand: "" };
            blocks.push(currentBlock);
            currentBlock = createBlock(++blockId);
        }
    });
    
    if (currentBlock.lines.length > 0 && !blocks.includes(currentBlock)) {
        blocks.push(currentBlock);
    }
    
    return blocks;
}
\end{lstlisting}

\subsection{Stap 3: Visualisatie}

%simpele uitleg

\begin{lstlisting}[language=JavaScript, caption={De conversie naar mermaid.js syntax.}]
function blocksToMermaid(blocks) {
    let mermaid = "%%{init: {'themeVariables': {'fontFamily': 'monospace'}}}%%\nflowchart TD\n";
    
    blocks.forEach(block => {
        const text = block.lines
        .map(l => l.raw.replace(/\s+/, ' ').trim()).join("<br/>");
        mermaid += `    ${block.id}["${text}"]\n`;
    });
    
    blocks.forEach((block, i) => {
        const next = blocks[i + 1];
        const exit = block.exit;
        
        // we have reached the last block
        if (!exit) return;
        
        const m = exit.mnemonic;
        const targetLabel = exit.operand;
        
        if (isConditionalBranch(m)) {
            const targetBlock = findBlockByLabel(blocks, targetLabel);
            if (targetBlock) mermaid += `    ${block.id} -->|${m}| ${targetBlock.id}\n`;
            if (next) mermaid += `    ${block.id} --> ${next.id}\n`;
        } else if (isUnconditionalBranch(m)) {
            const targetBlock = findBlockByLabel(blocks, targetLabel);
            if (targetBlock) mermaid += `    ${block.id} --> ${targetBlock.id}\n`;
        } else if (m === "SEQUENTIAL" && next) {
            mermaid += `    ${block.id} --> ${next.id}\n`;
        }
    });
    
    return mermaid;
}
\end{lstlisting}

\subsection{Gebruik van de applicatie}

Het gebruik van de tool verloopt in enkele eenvoudige stappen:

\begin{enumerate}

\section{Beperkingen van de implementatie}

De implementatie maakt bewust gebruik van een vereenvoudigde benadering van HLASM. Dit brengt een aantal beperkingen met zich mee.

Ten eerste wordt geen volledige syntactische of semantische analyse uitgevoerd. De parser werkt op basis van patroonherkenning en veronderstelt dat de invoer de standaard kolomstructuur volgt. Afwijkingen hiervan kunnen leiden tot foutieve interpretaties.

Daarnaast worden macro-instructies niet geëxpandeerd. Aangezien macros meerdere instructies kunnen bevatten, kan dit resulteren in een onvolledige weergave van de control-flow.

Een belangrijke beperking is de behandeling van vertakkingen. Enkel directe sprongen naar labels worden ondersteund. Indirecte sprongen, zoals via registers (bijvoorbeeld \texttt{BR R14}), kunnen niet statisch geanalyseerd worden en worden daarom niet opgenomen in de visualisatie.

Verder wordt de condition code niet inhoudelijk geïnterpreteerd. De tool detecteert enkel dat een conditionele vertakking aanwezig is, zonder de onderliggende logica te analyseren.

Deze beperkingen zijn aanvaardbaar binnen de context van deze proof-of-concept. De focus ligt op het zichtbaar maken van de globale structuur, niet op een volledige simulatie van het programma.

\section{Evaluatie}

De evaluatie heeft als doel te bepalen in welke mate de proof-of-concept bijdraagt aan het begrijpen van de control-flow in HLASM-programma’s.

Alle deelnemers hadden geen voorafgaande kennis van HLASM. Voorafgaand aan de test kregen zij een korte introductie over de basisprincipes van control-flow, met nadruk op labels en vertakkingen. Deze introductie was bewust beperkt om de deelnemers als beginnende gebruikers te behouden.

De evaluatie werd uitgevoerd met vier deelnemers. Elke deelnemer kreeg twee HLASM-codefragmenten en werd gevraagd om:

\begin{itemize}
    \item het startpunt te identificeren,
    \item vertakkingen aan te duiden,
    \item de globale structuur te beschrijven.
\end{itemize}

Deze opdrachten werden uitgevoerd:
\begin{itemize}
    \item zonder hulpmiddelen,
    \item met gebruik van de visualisatietool.
\end{itemize}

Zowel de benodigde tijd als de correctheid van de analyse werden geregistreerd.

\subsubsection{Kandidaat 1}

Kandidaat 1 had geen ervaring met HLASM en had moeite met het manueel volgen van vertakkingen. De analyse was traag en bevatte fouten.

Met de tool kon de kandidaat de structuur sneller en correcter identificeren.

\textbf{Bevinding}: duidelijke verbetering in snelheid en correctheid.

\subsubsection{Kandidaat 2}

Kandidaat 2 begreep de basisstructuur manueel, maar had moeite met complexere vertakkingen.

De visualisatie werd gebruikt als bevestiging van de interpretatie.

\textbf{Bevinding}: vermindering van onzekerheid en snellere validatie.

\subsubsection{Kandidaat 3}

Kandidaat 3 werkte systematisch en tekende de control-flow manueel uit. Dit was correct maar tijdrovend.

De tool verving dit proces volledig.

\textbf{Bevinding}: aanzienlijke efficiëntiewinst.

\subsubsection{Kandidaat 4}

Kandidaat 4 had de meeste moeite met manuele analyse en maakte meerdere fouten.

De visualisatie maakte de structuur duidelijker en hielp fouten corrigeren.

\textbf{Bevinding}: grootste impact bij minder ervaren gebruikers.

\subsection{Samenvatting van de resultaten}

Over alle deelnemers heen werden volgende trends vastgesteld:

\begin{itemize}
    \item De analysetijd daalde bij gebruik van de tool.
    \item De correctheid van de interpretatie nam toe.
    \item De programmastructuur werd als duidelijker ervaren.
\end{itemize}

Deze resultaten ondersteunen de hypothese dat een visuele representatie van control-flow het leerproces van HLASM versnelt, in het bijzonder voor beginnende ontwikkelaars.
